# NFL Capability Checklist Planning
**Date:** 2026-04-02
**Time:** 4:32–5:04 PM EST
**Attendees:** Kieran McConnell, Robert Treiber III

## Discussion

**Checklist-Driven Approach**
Kieran proposed shifting away from iterating on existing questions and instead building a baseline capability checklist per layer first, then deriving questions from it. Rationale: avoids "wheel spinning" (generating questions that don't map to anything real), ensures coverage, and keeps question development grounded in what clubs actually have or don't have.

Reference: Kieran has used this approach before — AI-generated initial list, refined through workshops, irrelevant items deleted. Clean and fast.

**Scoring Methodology Sequencing**
Kieran flagged confusion about how pre-game/in-game/post-game value weightings would apply to individual questions. Robert clarified that scoring weights (e.g., Wi-Fi 6 is mostly in-game) come after questions are finalized — the data viz team implements that logic. Both agreed: questions first, scoring later.

**TI Baseline Capabilities Identified**
Initial list discussed and refined during session:
- High-density Wi-Fi / access points
- DAS (Distributed Antenna System)
- Small cells
- Network backbone / fiber
- Core network switching
- QoS enforcement
- Redundant uplinks / failover circuits
- Edge compute
- Camera network
- Digital signage (hardware/physical)
- IoT sensors

Excluded from baseline: POS terminals, self-service kiosks — these are use cases, not foundational infrastructure.

**Digital Infrastructure Layer**
Agreed to run the same checklist process for DI. Target: 10–12 items per layer, from which ~10 survey questions are derived.

**Digital Signage Classification**
Confirmed: digital signage = physical/hardware infrastructure → DI layer (physical endpoints), not TS.

**Hardware Refresh Cycles**
Raised as a potential question dimension — linking screen generations and hardware age to revenue per fan and operational maturity. Needs framing work before it becomes a question.

## Decisions

1. **Checklist-first, questions-second** — build capability checklist per layer before drafting any questions — Kieran + Robert
2. **Defer scoring methodology** — scoring logic is a data viz team concern after questions are locked — Kieran + Robert
3. **TI baseline capabilities scoped** — 11-item list above; excludes use cases and software features
4. **Digital signage = DI (physical endpoints)** — confirmed classification
5. **Target: 10–12 checklist items → 10 questions per layer** — sizing for John Noble review
